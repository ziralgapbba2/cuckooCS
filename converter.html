📖 데이터 적용 가이드

변환된 JSON 파일을 대시보드에 적용하는 방법입니다.

📁 Step 1: 파일 배치

다운로드 받은 파일을 아래 구조로 배치하세요:

📁 dashboard/
├── index.html                    ← 서비스 접수 비교 분석
├── symptom_analysis.html         ← 문의내용 증상 분석
├── processing_dashboard.html     ← 처리건 분석
├── converter.html                ← 이 변환 도구
├── chart.umd.js                  ← Chart.js 라이브러리
├── manifest.json                 ← 📌 기간 목록 (다운로드 교체)
└── 📁 data/
    ├── 2026-01-reception.json    ← 접수 데이터
    ├── 2026-01-symptom.json      ← 증상 데이터
    ├── 2026-01-processing.json   ← 처리 데이터
    ├── 2026-02-reception.json    ← 매월 추가
    └── ...

📌 Step 2: manifest.json 교체

다운로드 받은 manifest.json으로 기존 파일을 교체(덮어쓰기)하세요.

manifest.json은 모든 대시보드가 공유하는 기간 목록입니다. 변환할 때마다 자동으로 새 기간이 추가됩니다.

⚠️ 기존 manifest.json이 있다면, 변환 전에 이 도구의 "manifest 불러오기" 기능으로 먼저 로드하세요.

🔄 Step 3: 데이터 JSON 복사

다운로드 받은 데이터 JSON을 data/ 폴더에 넣으세요.

  • 접수 분석: data/2026-01-reception.json
  • 증상 분석: data/2026-01-symptom.json
  • 처리 분석: data/2026-01-processing.json

✅ Step 4: 확인

브라우저에서 HTML 파일을 열고 기간 선택 드롭다운에서 새 기간이 표시되는지 확인하세요.

GitHub Pages 배포 시: git add . && git commit -m "2026-02 데이터 추가" && git push

💡 매월 업데이트 워크플로우

  1. 이 도구(converter.html)를 브라우저에서 열기
  2. 대시보드 유형 선택 (접수/증상/처리)
  3. 새 달의 엑셀 파일 드래그 & 드롭
  4. 컬럼 매핑 확인 → "변환 실행"
  5. 데이터 JSON + manifest.json 다운로드
  6. data/ 폴더에 JSON 복사, manifest.json 교체
  7. 3개 대시보드 유형에 대해 반복

🔧 기존 HTML에서 데이터 추출하기 (최초 1회)

이미 동작하는 기존 HTML이 있다면 브라우저 콘솔(F12)에서 데이터를 추출할 수 있습니다:

// index.html (접수 비교 분석)
copy(JSON.stringify(DASHBOARD_DATA))
→ 붙여넣기 후 data/2026-01-reception.json으로 저장

// symptom_analysis.html (증상 분석)
copy(JSON.stringify({meta:{period:"2026-01",totalRecords:78036},products:Object.keys(data),data:data}))
→ 붙여넣기 후 data/2026-01-symptom.json으로 저장

// processing_dashboard.html (처리건 분석)
// ※ 페이지 로딩 완료 후 실행 (rawData가 채워진 후)
copy(JSON.stringify({meta:{period:"2026-01"},rows:rawData}))
→ 붙여넣기 후 data/2026-01-processing.json으로 저장
